Архитектура OAuth-потока
# Strava OAuth 2.0 flow 1. PWA → GET /auth/strava Redirect to strava.com/oauth/authorize ?client_id=YOUR_ID &redirect_uri=https://streaklyapp.app/callback &scope=activity:read_all 2. Strava → POST /callback?code=ABC Server exchanges code for access_token Saves token encrypted in PostgreSQL 3. Strava Webhook → POST /webhooks/strava {event: "activity", owner_id: 12345} Server fetches activity, auto-checks habit ✅
⚠️ Текущая версия симулирует OAuth — реальные токены требуют бэкенда с HTTPS. OAuth-секреты НЕЛЬЗЯ хранить в браузере.
